This TryHackMe room was an easy, straight-forward room that explores the scenario of a L3 (level 3) analyst investigating escalations from a a L1 analyst.

The official scenario goes as follows:

“You are an SOC analyst on the SOC team at Managed Server Provider TrySecureMe. Today, you are supporting an L3 analyst in investigating flagged IPs, hashes, URLs, or domains as part of IR activities. One of the L1 analysts flagged two suspicious findings early in the morning and escalated them. Your task is to analyse these findings further and distil the information into usable threat intelligence.”

flagged IP address is: 101[.]99[.]76[.]120

flagged SHA256 hash: 5d0509f68a9b7c415a726be75a078180e3f02e59866f193b0a99eee8e39c874f

The desktop environment spawned from the target virtual machine has an app on the desktop called TryDetectThis2.0 and it is simply a self hosted version of virustotal.

First question: What is the name of the file identified with the flagged SHA256 hash?

Explanation: After looking up the hash in virustotal.com we can see that the filename is syshelpers.exe

Second question: What is the file type associated with the flagged SHA256 hash?

Answer: Win32 EXE

Explanation: It says it right under the filename

Third question: What are the execution parents of the flagged hash? List the names chronologically, using a comma as a separator. Note down the hashes for later use.

Answer: 361GJX7J,installer.exe

Explanation: Execution Parents are files that have been seen dropping the file in question (aka they’re droppers for the file in question). To the right of the dot it tells us the type of file and to the right it tells us the name of the file.

 

Fourth question: What is the name of the file being dropped? Note down the hash value for later use.

Answer: Aclient.exe

Explanation: In the Dropped Files section of this we can see various files that the file in question has in one way or another dropped onto the system. We can see that this particular file hasn’t had its file type identified yet.

Fifth question: Research the second hash in question 3 and list the four malicious dropped files in the order they appear (from up to down), separated by commas.

Analysis of the room

        From what I understand about realistic SOC roles, and from what people have told me, this seems pretty realistic. When a phishing incident happens such as the one in this room, where an employee gets phished and their machine gets compromised, the first step (after isolation) is always to get a sense of the scope, and to get a sense of the scope we first need to identify some critical details such as what caused the incident? If it’s malware then we move onto what it is attempting to do? Where did it come from? What ip is it communicating with? What is it associated with? What likely happened here is that an employee at TrySecureMe got a discord invite link and clicked it but it was actually a link that employed sophisticated phishing techniques to get that user to download and run malware. To deep dive into this case checkpoints research goes very in-depth on it.
